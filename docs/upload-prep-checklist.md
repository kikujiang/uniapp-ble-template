# 上传前整理清单（可复用）

此清单用于所有后续模板项目发布前检查。

## 1. 敏感信息

- `manifest.json` 中小程序 `appid` 使用占位值。
- 网络层 `baseURL` 使用模板配置文件，不写真实线上地址。
- 清理 token、key、secret、手机号、测试账号、设备编号等敏感信息。
- 删除或替换内部 URL、内部文档链接、内部沟通注释。

## 2. 忽略规则

- 确认 `.gitignore` 已覆盖：
  - `node_modules/`
  - `unpackage/`
  - `dist/`
  - `build/`
  - `logs/`
  - IDE 本地配置（如 `.hbuilderx/`、`.vscode/`、`.idea/`）
  - 系统临时文件（如 `.DS_Store`、`*.tmp`、`*.temp`）

## 3. 文档一致性

- README 与当前目录结构一致。
- 关键文档（模块接入、运行链路、绑定骨架）与现状一致。
- 明确“已完成”与“待完善”边界，避免误导。

## 4. 开源声明

- 根目录包含 `LICENSE`。
- 文档包含对外免责声明，并在 README 提供入口。

## 5. 发布动作

- 如需对外全新仓库，重新初始化 git 历史。
- 首次提交前执行最小可运行检查（至少启动与关键页面链路验证）。

## 6. 首个 commit 前真机 BLE 回归（必做）

- 回归范围：`bushuqi` 与 `cockroach` 双模块。
- 首页扫描链路：
  - 进入首页后可正常开启扫描。
  - 设备列表可被广播更新（在线态/最近发现时间）。
  - 离开页面后扫描监听被释放，无重复监听。
- 详情连接链路：
  - 进入详情页后可连接设备并开启 notify。
  - 可发送状态查询命令并收到可解析响应。
  - 页面退出后连接监听与 notify 资源正确释放。
- 绑定新增链路：
  - 进入绑定页后可扫描到可绑定设备。
  - 已绑定设备可被过滤，不重复展示。
  - 提交绑定后返回流程正常，列表可见新增结果。
- 平台覆盖建议：至少真机验证 iOS 1 台 + Android 1 台。
- 验收结论建议记录在 PR/提交说明中：通过/失败项、设备型号、系统版本、固件版本。
